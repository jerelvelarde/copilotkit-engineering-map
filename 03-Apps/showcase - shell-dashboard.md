# showcase - shell-dashboard

Internal-only **ops + parity dashboard** (`@copilotkit/showcase-shell-dashboard`, private, **port 3002**). A Next.js 15 / React 19 app that renders a **feature × integration grid** with live probe status, parity tiers, and reference-depth overlays. **Does not use CopilotKit** — it is a monitoring UI over the [[showcase - harness]] ops API.

## What it shows

Two-tab layout driven by overlay toggles (see `src/app/page.tsx`):

- **Matrix tab** — composable overlay layers over `catalog.json` (generated): feature support per integration, parity badges/tiers, reference-depth chips, docs reachability. Core pieces: `feature-grid`, `unified-cell`, `cell-model` (`buildCellModel`), `overlay-toggle-bar`, `adaptive-stats-bar`/`adaptive-legend`, `baseline-*` (baseline diffing), `parity-matrix`/`parity-badge`, `depth-chip`/`ref-depth-column`.
- **Status (Ops) tab** — probe schedule + live status grid: `status-tab`, `status-table`, `status-detail-panel`, `status-runs-list`, `status-duration-sparkline`, `status-trigger-button` (operator-token-gated manual trigger).

## Data + live wiring (verified)

- **Static catalog:** `catalog.json` is generated by `generate-registry.ts` (run via `predev`/`prebuild`/`pretest`) and cast to `CatalogData` (`src/data/catalog-types.ts`). `probe-docs.ts` also runs in those hooks → `docs-status.json` (per-feature docs reachability).
- **Live status:** `useLiveStatus()` opens a **single** PocketBase realtime subscription for all dimensions (deliberately one connection to avoid a 7-way SSE race), merged via `mergeRowsToMap` (`src/lib/live-status.ts`). PocketBase client in `src/lib/pb.ts` (`pocketbase` SDK dependency).
- **Probe schedule + triggers:** `useProbes()` polls the ops API every 10s; `useTriggerProbe({ token })` POSTs `/api/ops/trigger` with the operator token (`NEXT_PUBLIC_OPS_TRIGGER_TOKEN`); fails loud if the token is unset (`src/hooks/use-probes.ts`, `src/lib/ops-api.ts`).
- **Same-origin proxy to harness:** `next.config.ts` rewrites `/api/ops/:path*` → `${OPS_BASE_URL}/api/:path*`. `OPS_BASE_URL` is **required at build/start** (the config throws otherwise) — this sidesteps [[showcase - harness]]'s lack of a CORS allowlist and keeps the ops base URL out of the client bundle.

## Tests

- **Unit/component:** Vitest + Testing Library (`vitest.config.ts`, jsdom) — extensive `__tests__/` coverage for cells, overlays, status panels, hooks (`useBaseline`, `useOverlays`, `useLiveStatus`, `useLastTransition`).
- **Visual:** Playwright (`playwright.config.ts`, `tests/visual/dashboard.spec.ts`).
- **Seed scripts:** `scripts/generate-seed.ts`, `scripts/seed-baseline.ts` produce `src/data/baseline-seed.json`.

## Build / deploy

- **Bundler:** Next.js with Turbopack in dev (`next dev --turbopack --port 3002`).
- **Docker:** `shell-dashboard/Dockerfile`. Deployed to Railway as an internal dashboard (default host referenced as `https://dashboard.showcase.copilotkit.ai`).
