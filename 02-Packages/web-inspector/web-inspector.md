# @copilotkit/web-inspector

A framework-agnostic, **Lit-based web component** that overlays a floating CopilotKit inspector/devtools panel on any page. Published as **`@copilotkit/web-inspector`** at **v1.57.4** (public). This overview doubles as the folder MOC.

It attaches to a [[core - CopilotKitCore]] instance, subscribes to its runtime/agent/context/thread-store changes, records the live [[AG-UI Protocol]] event stream per agent, and renders panels for AG-UI events, agents, frontend tools, context, threads, and settings. See [[Debug Mode]] for the broader debugging story.

> [!note] Lit, not Stencil
> The package ships a leftover `src/components.d.ts` autogenerated stub that references `@stencil/core`, but it is **empty and unused** — the real implementation is Lit. Ignore the Stencil typings.

## Entry points / exports

- `package.json` `exports["."]` → `dist/index.mjs` / `dist/index.cjs` (+ `main`/`module`/`types`, and a UMD build `dist/index.umd.js` global `CopilotKitWebInspector` with `lit`, `marked`, `lucide`, `@copilotkit/core` externalized).
- `src/index.ts` is the single implementation module. It exports `WEB_INSPECTOR_TAG` (`"cpk-web-inspector"`), the classes `WebInspectorElement` and `ɵCpkThreadDetails`, and `defineWebInspector()`.
- **Self-registers on import:** `index.ts` calls `defineWebInspector()` at module load, registering `<cpk-web-inspector>`, plus `<cpk-thread-list>` and `<cpk-thread-details>` (each guarded by `customElements.get(...)`). A `declare global` augments `HTMLElementTagNameMap["cpk-web-inspector"]`.

## Usage

Import the package (registers the element) and drop the tag on the page; set its `core` property (or rely on `auto-attach-core`):

```ts
import "@copilotkit/web-inspector";
const el = document.querySelector("cpk-web-inspector");
el.core = myCopilotKitCore; // a CopilotKitCore instance
```

`auto-attach-core` (default true) makes the element discover a core from `window.__COPILOTKIT_CORE__`, `window.copilotkit.core`, or `window.copilotkitCore`.

## Custom elements / notes in this folder

- [[web-inspector - cpk-web-inspector (Lit)]] — the main element (`WebInspectorElement`): floating button + draggable/resizable/dockable window, core+agent subscriptions, AG-UI event capture, and all panels.
- [[web-inspector - cpk-thread-list]] — `<cpk-thread-list>` (`CpkThreadList`): searchable thread sidebar.
- [[web-inspector - cpk-thread-details]] — `<cpk-thread-details>` (`ɵCpkThreadDetails`): per-thread Conversation / Agent State / AG-UI Events tabs, fetched from the runtime's `/threads/:id/*` endpoints.

## Internal libs / notes in this folder

- [[web-inspector - context-helpers]] — pure geometry helpers for positioning/anchoring/clamping the floating button and window.
- [[web-inspector - persistence]] — localStorage (with legacy-cookie migration) for window/dock state, plus the anonymous telemetry distinct-ID + opt-out flags.
- [[web-inspector - telemetry]] — fire-and-forget anonymous funnel events (banner viewed/clicked, threads-tab clicked).
- [[web-inspector - event color/types]] — the inspector's AG-UI event-type union, per-type color mapping, and JSON syntax highlighter.

## Depends on / depended on by

- **Depends on:** [[@copilotkit/core]] (`workspace:*`) for `CopilotKitCore`, its subscriber type, and the ɵ thread-store internals; `@ag-ui/client` (`AbstractAgent`, `AgentSubscriber`) for the [[AG-UI Protocol]] event stream; `lit` (`^3.2.0`); `lucide` (icons); `marked` (announcement banner markdown).
- **Depended on by:** dev/showcase surfaces that embed the inspector; opt-in for any app.

## Build / test

- **Bundler:** tsdown (ESM+CJS unbundled + UMD), with `.css` loaded as `text` and `.svg` as `dataurl`.
- **CSS:** Tailwind **v4** CLI compiles `src/styles/tailwind.css` → `src/styles/generated.css`, which is imported as a string and injected into the Lit shadow DOM (it does **not** use the shared [[tailwind-config]] v3 preset).
- **Test:** vitest with the `jsdom` environment (`src/__tests__/`, `src/lib/__tests__/`).
- **TS config:** `extends @copilotkit/typescript-config/base.json` plus `experimentalDecorators` + `useDefineForClassFields: false` for Lit — see [[typescript-config]].
