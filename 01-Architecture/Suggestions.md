# Suggestions

**Suggestions** are the clickable "next message" prompts shown in the chat UI. They are computed entirely on the frontend by the [[core - SuggestionEngine]] inside [[core - CopilotKitCore]] (`packages/core/src/core/suggestion-engine.ts`); the types are [[core - Suggestion types]] (`packages/core/src/types.ts`).

## Two kinds

`SuggestionsConfig` is a union:
- **Static** (`StaticSuggestionsConfig`) ‚Äî a fixed list of `{ title, message }` shown verbatim.
- **Dynamic** (`DynamicSuggestionsConfig`) ‚Äî generated by the model from `instructions`, bounded by `minSuggestions`/`maxSuggestions`. The engine asks a provider agent to produce them, so dynamic suggestions are themselves a small agent run.

Both carry an `available` window ‚Äî `"before-first-message" | "after-first-message" | "always" | "disabled"` ‚Äî and agent scoping: `providerAgentId` (who generates) and `consumerAgentId` (which chat shows them, default `"*"`). This makes suggestions correct in [[Multi-Agent]] setups.

## Engine API (on the core)

```ts
addSuggestionsConfig(config): string   // returns an id
removeSuggestionsConfig(id): void
reloadSuggestions(agentId): void       // clear + regenerate
clearSuggestions(agentId): void
getSuggestions(agentId): { suggestions, isLoading }
```

The engine emits `onSuggestionsChanged`, `onSuggestionsStartedLoading`, and `onSuggestionsFinishedLoading` to subscribers, and each `Suggestion` carries `isLoading` so the UI can render placeholders while a dynamic batch streams in.

## Framework bindings

- React V2: `useSuggestions` ([[react-core - useSuggestions]]); UI in [[react-ui - Suggestions]].
- Vue: composables ([[vue - composables (suggestions/interrupt/threads/‚Ä¶)]]).
- Angular surfaces them through its chat components ([[angular - Chat components]]).

Suggestions reuse the same agent/run machinery and [[AG-UI Protocol]] as a normal turn; they are not a separate server endpoint.
